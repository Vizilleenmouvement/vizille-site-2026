Guide d'administration

Vizille en Mouvement - Comment mettre à jour le site

1. Se connecter

1
Allez sur admin.html (ou cliquez sur ⚙️ dans le menu du site)
2
Entrez le mot de passe administrateur
3
Entrez votre Token GitHub en haut à droite (obligatoire pour publier)
Token GitHub : C'est une clé qui vous autorise à modifier le site. Voir la section "Créer un Token GitHub" en fin de guide.

2. Ajouter un candidat

1
Cliquez sur l'onglet 👥 Candidats
2
Cliquez sur + Nouveau candidat
3
Remplissez : Nom, Prénom, Rôle, Bio, Photo
4
Cliquez sur Enregistrer
5
Cliquez sur 📤 Publier

3. Ajouter un projet

1
Cliquez sur l'onglet 🚀 Projets
2
Cliquez sur + Nouveau projet
3
Remplissez les champs :
  • Titre : nom du projet
  • Thème(s) : cliquez sur les badges pour sélectionner
  • Statut : Programmé, En cours, Réalisé
  • Résumé : courte description
  • Description : texte complet
4
Cliquez sur Enregistrer puis 📤 Publier

4. Ajouter une actualité

1
Cliquez sur l'onglet 📰 Articles
2
Cliquez sur + Nouvel article
3
Remplissez :
  • Titre : titre de l'article
  • Date : date de publication
  • Catégorie : Actualité, Événement, Communiqué...
  • Extrait : résumé court
  • Contenu : texte complet
4
Enregistrer puis 📤 Publier

5. Répondre à une question

1
Cliquez sur l'onglet 🤝 Construisons ensemble
2
Cliquez sur + Nouvelle question
3
Remplissez :
  • Question : la question posée
  • Réponse : votre réponse
  • Auteur : prénom (optionnel)
4
Enregistrer puis 📤 Publier

6. Ajouter une action au bilan

1
Cliquez sur l'onglet 📊 Actions
2
Cliquez sur + Nouvelle action
3
Remplissez : Titre, Thème, Statut (Réalisé/En cours/Prévu), Description
4
Enregistrer puis 📤 Publier

⚠️ Règles importantes

Toujours PUBLIER après avoir enregistré !
Si vous ne cliquez pas sur "Publier", vos modifications ne seront pas visibles sur le site.

Les indicateurs de statut

  • ✅ À jour : tout est publié
  • ⚠️ Modifié : des modifications attendent d'être publiées
  • ⏳ ... : chargement en cours
  • ❌ Erreur : vérifiez votre token GitHub

Délai de mise à jour

Après publication, attendez 1 à 5 minutes pour voir les changements sur le site. C'est normal !

7. Créer un Token GitHub

Le token est une clé de sécurité qui vous permet de publier sur le site.

🔐 Identifiants GitHub :
Compte : Vizilleenmouvement
Mot de passe : (demander à l'administrateur)
1
Allez sur github.com et connectez-vous avec les identifiants ci-dessus
2
Cliquez sur votre photo de profil (en haut à droite) → Settings
3
Dans le menu à gauche, tout en bas, cliquez sur Developer settings
4
Cliquez sur Personal access tokens → Tokens (classic)
5
Cliquez sur Generate new token → Generate new token (classic)
6
Remplissez :
  • Note : Admin Vizille
  • Expiration : No expiration (pour ne pas avoir à le refaire)
  • Cochez : ☑️ repo (accès aux fichiers)
7
Cliquez sur Generate token en bas de page
8
Copiez le token (il commence par ghp_...) et collez-le dans l'admin
Attention ! Le token ne s'affiche qu'une seule fois. Copiez-le immédiatement ! Si vous le perdez, il faudra en créer un nouveau.

🔗 Accéder à GitHub Tokens

8. Inviter un collaborateur

Pour donner accès à une autre personne sans partager vos identifiants :

1
Allez sur github.com/Vizilleenmouvement/Vizille-en-mouvement
2
Cliquez sur Settings (onglet en haut)
3
Dans le menu à gauche, cliquez sur Collaborators
4
Cliquez sur Add people
5
Entrez l'email de la personne à inviter
Ce qui se passe ensuite :
La personne reçoit un email d'invitation. Elle devra créer un compte GitHub gratuit (si elle n'en a pas), puis créer son propre token en suivant la section 7 de ce guide.

🔗 Gérer les collaborateurs

9. En cas de problème

  • Vérifiez que le token GitHub est bien entré
  • Rafraîchissez la page (F5 ou Cmd+R)
  • Attendez quelques minutes (cache GitHub)
  • Contactez l'administrateur technique
